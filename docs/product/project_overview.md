# プロジェクト概要 - Ad-Insight-Spec

## 1. プロジェクト目的

既存のYouTube動画分析システム「video-insight-spec（VIS）」のアーキテクチャおよびソースコード資産を最大限流用し、Web広告（まずはMeta広告）およびランディングページ（LP）の統合分析・レポートサービスを迅速に立ち上げる。

## 2. 背景・課題

### 既存資産の活用可能性
VISで培った「非構造化コンテンツからの知識抽出」「LLMによる構造化JSON（insight_spec）の生成」「ダッシュボード・レポートの自動生成」というパイプラインは、Web広告領域にも強力に適用可能である。

### Web広告分析の重要性
- 広告代理店や事業会社のマーケターが、勝ちクリエイティブの要因分析に時間をかけている
- クリエイティブ疲弊の検知が遅れ、無駄な広告費が増加している
- ランディングページ（LP）と広告のメッセージ一貫性を定量的に評価する手段がない

## 3. 対象ユーザー

- **広告代理店のマーケター**：複数のクライアント広告キャンペーンを効率的に分析したい
- **事業会社のマーケティング担当者**：自社の広告パフォーマンスを詳細に把握したい
- **経営陣**：広告投資のROI改善施策を迅速に意思決定したい

## 4. 初期スコープ（Phase 0～1）

### 対応媒体
- **Meta広告**（Facebook/Instagram）
  - 画像クリエイティブ
  - 動画クリエイティブ
  - テキスト・見出し・CTA
  - キャンペーン・広告セット・フォーマット・プレースメント情報

### 分析対象データ
- **広告メタデータ**：広告ID、キャンペーン名、広告セット名、分析期間
- **クリエイティブ要素**：フォーマット、プライマリテキスト、見出し、CTA
- **ランディングページ**：遷移先URL、FV訴求、オファー内容、フォーム難易度
- **配信実績・KPI**：インプレッション、クリック数、CTR、CPA、CVR、消化金額

### 初期機能
1. **ポートフォリオビュー**：全広告のCPA/CVRマッピング
2. **クリエイティブ詳細分析**：フック、ボディ、CTAの要素分解と評価
3. **広告×LPの一貫性診断**：メッセージのズレによる離脱リスク警告
4. **AI改善提案**：実績データに基づく具体的な改善アクション

## 5. 将来展開（Phase 3 以降）

データ構造は将来的にGoogle広告、TikTok広告等への展開（クロスプラットフォーム化）を前提に設計する。

## 6. 成功指標

- **開発効率**：VIS資産の流用により、初期実装を60日以内に完了
- **分析精度**：AIが検出したクリエイティブ改善提案の実装率 > 60%
- **ユーザー満足度**：マーケター向けの「次に何をすべきか」が明確に提示される

## 7. 次に決めるべき論点

### (a) 評価KPIの優先順位
ダッシュボードのトップに置く絶対的KPIはCPAか、ROASか、あるいは別の指標か。

**影響**: UI/UXレイアウト、データ集計ロジック、アラート機構

### (b) 診断の粒度
LLMによるクリエイティブ要素分解（フック・ボディ・CTA）はどこまで細かく行うか。

**影響**: LLMプロンプト設計、DB スキーマ、分析精度

### (c) 広告とLPの結合ルール
URLパラメータ（UTM等）で結合するのか、APIの階層構造を正として紐付けるのか。

**影響**: データ取得パイプライン、ジョイン処理の信頼性

### (d) データ取得の自動化範囲
初期フェーズはCSV手動インポートとするか、最初からMeta Graph API連携を作り込むか。

**影響**: スケジューラ設計、API認証管理、運用負荷

### (e) 媒体抽象化のレベル
最初から「動画」と「静止画」のスキーマを完全に分けるか、統合フォーマットで処理するか。

**影響**: JSONスキーマ設計、将来の拡張性

### (f) 最初の営業用デモ範囲
PoCとして顧客に見せるための「キラー分析機能（例：隠れたLP離脱要因の発見）」はどれに絞るか。

**影響**: Phase 1の優先順位、UI実装範囲
